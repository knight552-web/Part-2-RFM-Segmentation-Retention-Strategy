# D2C Customer Churn – Part 2: RFM Segmentation & Retention Strategy

## Overview
This repository contains Part 2 of the D2C Customer Churn Intelligence Capstone Project.

Objectives:
- Build RFM-based customer segments
- Incorporate behavioral and support signals
- Design targeted retention actions
- Prioritize retention budget allocation
- Review ambiguous customer cases manually

## Repository Contents
- rfm_segmentation.ipynb
- segments.csv
- retention_strategy.md
- manual_review_cases.md
- requirements.txt

## Segmentation Inputs
- Recency
- Frequency
- Monetary Value
- Support ticket behavior
- Sentiment
- Return rate
- Discount dependency
- Web/app engagement
- Campaign history

## Key Segments
- Champions
- Loyal Customers
- At Risk Customers
- Dormant Customers
- High-Value Unhappy Customers
- Discount Sensitive Customers
- Browsers Not Buyers
- New Customers

## How to Run
1. Install requirements
2. Open rfm_segmentation.ipynb
3. Execute notebook cells
4. Export segments.csv

## Deliverables
- Segmentation logic
- Segment-level analysis
- Retention strategy recommendations
- Budget prioritization
- Manual review cases
